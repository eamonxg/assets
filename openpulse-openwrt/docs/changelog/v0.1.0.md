## v0.1.0 — Initial Release

First release of openpulse-server for OpenWrt.

### Features

- REST and SSE API for OpenPulse Mobile — system info, network, DNS, IPv6, speedtest, logs, live metrics stream
- Statically-linked musl binary — no runtime dependencies, no libc, no libstdc++
- Token-based Bearer auth with per-IP rate limiting (replaces global lockout)
- Modular architecture: each domain area is an independent `ServiceModule`
- Built-in `/api/v1/health` and `/api/v1/modules` introspection endpoints
- `procd` managed service with auto-generated API token on first start
- UCI configuration at `/etc/config/openpulse` (port, token, enabled)

### Supported Architectures

| Package | Covers |
|---------|--------|
| `aarch64_generic` | Generic 64-bit ARM |
| `aarch64_cortex-a53` | Cortex-A53 (RPi 3/4, many SoCs) |
| `aarch64_cortex-a72` | Cortex-A72 (RPi 4 64-bit) |
| `aarch64_cortex-a76` | Cortex-A76 (RPi 5) |
| `x86_64` | x86-64 routers and VMs |
| `arm_cortex-a7` and variants | Cortex-A7/A8/A9/A15 (ipq40xx, mt7622, …) |

### Installation

```sh
# Download the .apk for your architecture, then:
apk add --allow-untrusted openpulse-server-0.1.0-r1.apk

# Or via ipk:
opkg install openpulse-server_0.1.0-1_<arch>.ipk

service openpulse start
uci get openpulse.config.token   # copy this token into OpenPulse Mobile
```
